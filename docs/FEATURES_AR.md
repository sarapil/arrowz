## 1. هاتف WebRTC الذكي

### 1.1 نظرة عامة
هاتف SIP كامل الميزات يعمل في المتصفح باستخدام مكتبة **JsSIP 3.x**، مدمج مباشرة في شريط التنقل Frappe للوصول الفوري من أي صفحة.

### 1.2 الميزات الأساسية

| الميزة | الوصف | الحالة |
|--------|--------|--------|
| **تكامل شريط التنقل** | هاتف منسدل يمكن الوصول إليه من أي صفحة | ✅ فعال |
| **عرض النافذة** | نافذة ملء الشاشة للجوال | ✅ فعال |
| **إجراء المكالمات** | الاتصال بأرقام خارجية وتحويلات داخلية | ✅ فعال |
| **استقبال المكالمات** | قبول/رفض المكالمات الواردة مع معرف المتصل | ✅ فعال |
| **تعليق المكالمة** | وضع المكالمات قيد الانتظار واستئنافها | ✅ فعال |
| **كتم الصوت** | كتم/إلغاء كتم الميكروفون أثناء المكالمة | ✅ فعال |
| **التحويل المباشر** | تحويل المكالمة مباشرة إلى تحويلة أخرى | ✅ فعال |
| **التحويل المُعلن** | استشارة قبل التحويل | ✅ فعال |
| **دعم DTMF** | لوحة أرقام للتنقل في IVR أثناء المكالمات | ✅ فعال |
| **التسجيل التلقائي** | تسجيل SIP تلقائي مع منطق إعادة المحاولة | ✅ فعال |
| **طلب الميكروفون المسبق** | طلب أذونات الميكروفون مسبقاً لاتصال أسرع | ✅ فعال |
| **مكالمات الفيديو** | دعم مكالمات الفيديو الاختيارية | ✅ فعال |

### 1.3 المواصفات التقنية

```yaml
المكتبة: JsSIP 3.x
البروتوكول: WebSocket SIP (WSS/WS)
الكوديك المدعومة: OPUS, G.722, PCMU (G.711μ), PCMA (G.711A)
ملف JavaScript: softphone_v2.js (2365 سطر)
مساحة الأسماء: arrowz.softphone
```

### 1.4 مكونات واجهة المستخدم

| المكون | الوصف |
|--------|--------|
| **مؤشر الحالة** | يعرض حالة التسجيل (مسجل، يتصل، غير متصل) |
| **محول التحويلات** | التبديل بين تحويلات متعددة (إذا تم تعيينها) |
| **لوحة الأرقام** | لوحة أرقام كاملة مع * و # |
| **بحث جهات الاتصال** | بحث إكمال تلقائي لجهات الاتصال/العملاء المحتملين |
| **مؤقت المكالمة** | عداد مدة مباشر أثناء المكالمات |
| **أزرار الإجراءات** | تعليق، كتم، تحويل، إنهاء، لوحة أرقام |
| **التحكم في مستوى الصوت** | ضبط مستوى صوت السماعة |
| **المكالمات الأخيرة** | وصول سريع للأرقام الأخيرة |

### 1.5 خيارات التكوين

| الخيار | الوصف | الموقع |
|--------|--------|--------|
| خادم WSS | عنوان خادم WebSocket SIP | AZ Server Config |
| نطاق SIP | نطاق تسجيل SIP | AZ Server Config |
| خادم STUN | خادم لاجتياز NAT | AZ Server Config |
| خادم TURN | خادم لاجتياز NAT مع الترحيل | AZ Server Config |
| التسجيل التلقائي | التسجيل تلقائياً عند تحميل الصفحة | لكل تحويلة |
| تمكين الفيديو | السماح بمكالمات الفيديو | لكل تحويلة |

### 1.6 دعم التحويلات المتعددة
يمكن للمستخدمين الحصول على تحويلات SIP متعددة والتبديل بينها ديناميكياً:

- **التحويلة الأساسية**: التحويلة الافتراضية للمكالمات الصادرة
- **التحويلات الثانوية**: تحويلات إضافية (مثل خطوط الأقسام)
- **محول التحويلات**: قائمة منسدلة لتغيير التحويلة النشطة
- **إعدادات لكل تحويلة**: يمكن أن يكون لكل تحويلة إعدادات مختلفة

### 1.7 نقاط API

| نقطة النهاية | الطريقة | الوصف |
|-------------|---------|--------|
| `arrowz.api.webrtc.get_server_config` | GET | الحصول على تكوين خادم SIP |
| `arrowz.api.webrtc.get_user_extension` | GET | الحصول على تحويلة المستخدم الحالي |
| `arrowz.api.webrtc.get_user_extensions` | GET | الحصول على جميع تحويلات المستخدم |
| `arrowz.api.webrtc.register_webrtc` | POST | تسجيل عميل WebRTC |
| `arrowz.api.webrtc.unregister_webrtc` | POST | إلغاء تسجيل عميل WebRTC |
| `arrowz.api.webrtc.make_call` | POST | بدء مكالمة صادرة |
| `arrowz.api.webrtc.hangup_call` | POST | إنهاء المكالمة النشطة |
| `arrowz.api.webrtc.transfer_call` | POST | تحويل المكالمة النشطة |
| `arrowz.api.webrtc.hold_call` | POST | تعليق/استئناف المكالمة |
| `arrowz.api.webrtc.send_dtmf` | POST | إرسال نغمات DTMF |

| الحقل | الوصف | تعبئة تلقائية |
|-------|--------|---------------|
| **الاتجاه** | وارد أو صادر | ✅ |

| **اسم جهة الاتصال** | محلول من CRM | ✅ |
| **وقت البدء** | وقت بدء المكالمة | ✅ |
| **وقت الرد** | وقت الرد على المكالمة | ✅ |
| **وقت الإنهاء** | وقت إنهاء المكالمة | ✅ |
| **المدة** | إجمالي مدة المكالمة (بالثواني) | ✅ |
| **مدة الرنين** | الوقت قبل الرد | ✅ |
| **النتيجة** | تم الرد، لا رد، مشغول، فشل، بريد صوتي | ✅ |
| **التحويلة** | التحويلة التي تعاملت مع المكالمة | ✅ |

| **مسار التسجيل** | مسار تسجيل المكالمة | ✅ |
| **رابط التسجيل** | رابط التشغيل | ✅ |
| **نوع طرف CRM** | عميل محتمل، عميل، جهة اتصال، مورد | ✅ |
| **طرف CRM** | رابط لمستند CRM | ✅ |
| **تصنيف المشاعر** | مشاعر محللة بالذكاء الاصطناعي | اختياري |
| **درجة المشاعر** | ثقة المشاعر (0-1) | اختياري |
| **النسخ النصي** | تحويل الصوت إلى نص | اختياري |

### 2.2 تسجيل المكالمات الصوتية

إدارة كاملة للتسجيل مع التشغيل المتدفق والتنزيل.

| الميزة | الوصف |
|--------|--------|
| **التشغيل المتدفق** | تشغيل التسجيلات في المتصفح |
| **التنزيل** | تنزيل كـ MP3/WAV |
| **التحكم في الأذونات** | وصول قائم على الأدوار للتسجيلات |
| **مسار قابل للتكوين** | مسار تخزين مخصص للتسجيل |
| **إمكانية الحذف** | إزالة التسجيلات القديمة |

**الصيغ المدعومة:** audio/mpeg, audio/wav, audio/ogg, audio/x-gsm

#### API التسجيل

| نقطة النهاية | الطريقة | الوصف |
|-------------|---------|--------|
| `arrowz.api.recording.stream_recording` | GET | بث صوت التسجيل |
| `arrowz.api.recording.download_recording` | GET | تنزيل ملف التسجيل |
| `arrowz.api.recording.get_recording_url` | GET | الحصول على رابط التشغيل |
| `arrowz.api.recording.delete_recording` | DELETE | حذف التسجيل |

### 2.3 تسجيل تحويل المكالمات

تتبع جميع تحويلات المكالمات مع التفاصيل:

| الحقل | الوصف |
|-------|--------|
| نوع التحويل | مباشر أو مُعلن |
| التحويلة المصدر | من بدأ التحويل |
| التحويلة الهدف | وجهة التحويل |
| الطابع الزمني | وقت حدوث التحويل |
| النتيجة | نجاح، فشل، رُفض |

### 2.4 إدارة نتائج المكالمات

| النتيجة | الوصف |
|---------|--------|
| **ANSWERED** | تم الرد على المكالمة وحدثت محادثة |
| **NO ANSWER** | انتهت مهلة الرنين، لا رد |
| **BUSY** | الوجهة مشغولة |
| **FAILED** | فشلت المكالمة (شبكة، رقم غير صالح، إلخ) |
| **VOICEMAIL** | ذهبت المكالمة إلى البريد الصوتي |
| **CANCELLED** | أنهى المتصل قبل الرد |
